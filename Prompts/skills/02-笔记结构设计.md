# 技能：笔记结构设计

## 核心原则

> **渐进式深入**：从直觉到数学，从 What 到 Why 到 How。

---

## 一、文件层级结构

### 推荐目录结构
```
NoteByHuman/
├── LectureX/
│   ├── LectureX-Main.md          # 主笔记，核心内容
│   ├── LectureX-[专题名].md      # 深度专题文档
│   ├── LectureX-[专题名]-Proof.md # 数学证明
│   ├── images/                   # 本地图片
│   └── code/                     # 代码示例 (可选)
```

### 文件命名规范
- 使用 `LectureX-` 前缀保持排序
- 专题名使用 **英文短横线连接**: `Speculative-Sampling`
- 避免空格和特殊字符

## 三、内容层级规范

### 3.1 深度分级

| 层级 | 目标读者 | 内容特点 |
|------|----------|----------|
| L1 直觉层 | 初学者 | 类比、图示、一句话总结 |
| L2 概念层 | 有基础者 | 定义、流程、关键公式 |
| L3 细节层 | 进阶者 | 完整推导、边界条件、实现细节 |
| L4 研究层 | 研究者 | 开放问题、前沿进展、批判分析 |

### 3.2 视觉层级

```markdown
# H1: 章节标题 (每个主题一个)
## H2: 小节标题 (核心概念)
### H3: 子主题 (技术细节)
#### H4: 细分点 (很少使用)

**粗体**: 关键术语首次出现
*斜体*: 强调、注释
`代码`: 函数名、变量名、命令
> 引用块: 重要洞见、一句话总结
```

## 四、内容组织模式

### 4.1 问题驱动模式 (推荐)
```markdown
## [问题名称]

**问题**: [描述问题]
**原因**: [为什么会有这个问题]
**解决方案**: [如何解决]
**效果**: [解决后的改进]
```

### 4.2 对比模式
```markdown
## A vs B

| 维度 | A | B |
|------|---|---|
| 原理 | ... | ... |
| 优点 | ... | ... |
| 缺点 | ... | ... |
| 适用场景 | ... | ... |
```

### 4.3 演进模式
```markdown
## 技术演进: A → B → C

### V1: A
- 特点: ...
- 问题: ...

### V2: B (改进 A)
- 改进点: ...
- 新问题: ...

### V3: C (改进 B)
- 最终形态: ...
```

## 六、质量检查清单

- [ ] **完整性**: 覆盖了课程的核心内容
- [ ] **准确性**: 公式、数字经过核对
- [ ] **可读性**: 有直觉解释，不只是公式堆砌
- [ ] **一致性**: 术语使用统一，格式规范
- [ ] **链接有效**: 所有图片和链接可访问
- [ ] **独立性**: 每个文档可独立阅读理解
